# R — FAI Standards

## Tidyverse Conventions

- Prefer `dplyr` verbs (`mutate`, `filter`, `summarise`, `select`, `arrange`) over base R subsetting
- Use native pipe `|>` for new code (R 4.1+); `%>%` acceptable in existing magrittr codebases
- `tidyr::pivot_longer` / `pivot_wider` over `reshape2`; `tidyr::unnest` for list-columns
- `purrr::map` family for typed iteration — `map_chr`, `map_dbl`, `map_dfr` over `sapply`/`lapply`
- `ggplot2` for all visualization — layer grammar, never mix base `plot()` with ggplot

```r
# Idiomatic tidyverse pipeline
results <- raw_data |>
  filter(!is.na(score), status == "active") |>
  mutate(score_z = (score - mean(score)) / sd(score)) |>
  group_by(region) |>
  summarise(
    mean_score = mean(score_z),
    n = n(),
    .groups = "drop"
  ) |>
  arrange(desc(mean_score))
```

## Function Design

- `snake_case` for all functions and variables — never camelCase or dot.case
- Mandatory default args for optional parameters; data arg first for pipe compatibility
- Use tidy evaluation `{{ }}` (curly-curly) for column name arguments — not `enquo`/`!!`
- Return early on invalid input with `rlang::abort()` and classed conditions

```r
summarise_by_group <- function(data, group_col, value_col, na_rm = TRUE) {
  checkmate::assert_data_frame(data, min.rows = 1)
  data |>
    group_by({{ group_col }}) |>
    summarise(
      mean_val = mean({{ value_col }}, na.rm = na_rm),
      sd_val   = sd({{ value_col }}, na.rm = na_rm),
      n        = n(),
      .groups  = "drop"
    )
}
```

## Package Structure

- `DESCRIPTION` — Title, Version, Imports (not Depends), Authors@R with ORCID
- `NAMESPACE` — generated by roxygen2, never hand-edit
- `R/` — one file per logical unit, no mega-files; use `R/utils.R` for internal helpers
- `tests/testthat/` — mirror `R/` structure; `test-<module>.R` naming
- `man/` — generated by `devtools::document()`, committed to repo
- `vignettes/` — long-form usage examples in Rmd/Quarto
- Run `devtools::check()` with zero errors/warnings before every release

## Testing with testthat

- `testthat::test_that()` with descriptive labels — one concept per test
- `expect_equal()` with `tolerance` for floating point; `expect_error(class = "my_error")`
- Snapshot tests (`expect_snapshot`) for ggplot outputs and printed summaries
- Use `withr::local_tempdir()` / `withr::local_options()` for side-effect isolation

```r
test_that("summarise_by_group computes correct means", {
  df <- tibble(grp = c("a", "a", "b"), val = c(10, 20, 30))
  result <- summarise_by_group(df, grp, val)
  expect_equal(result$mean_val, c(15, 30))
  expect_equal(nrow(result), 2)
})

test_that("summarise_by_group rejects empty data", {
  expect_error(
    summarise_by_group(tibble(), grp, val),
    class = "simpleError"
  )
})
```

## data.table for Performance

- Use `data.table` for datasets >1M rows — `:=` for in-place mutation, `.SD` for column ops
- Keys with `setkey()` for O(log n) joins; `fread`/`fwrite` for fast I/O
- Never mix `dplyr` and `data.table` syntax in the same pipeline — pick one per module

```r
library(data.table)
dt <- fread("large_dataset.csv")
setkey(dt, customer_id)
result <- dt[status == "active",
  .(mean_revenue = mean(revenue), count = .N),
  by = .(region, quarter)
][order(-mean_revenue)]
```

## Environment & Reproducibility

- `renv::init()` on every project — lock file committed, `renv/library` gitignored
- Pin R version in `.Rprofile` or Docker (`rocker/r-ver:4.4.0`)
- `renv::snapshot()` after any `install.packages()`; CI runs `renv::restore()`
- R Markdown / Quarto for reproducible reports — set `seed` in setup chunk

## Shiny Patterns

- Shiny modules (`moduleServer`) for every reusable UI/server pair — never monolith `app.R`
- `reactiveVal()` / `reactiveValues()` for mutable state; `bindEvent()` to debounce
- `req()` to guard against NULL inputs; `validate(need(...))` for user-facing messages
- Session-scoped cleanup with `session$onSessionEnded()`

## Error Handling

- `tryCatch()` for expected recoverable errors; `withCallingHandlers()` for logging + re-raise
- `rlang::abort(message, class = "pkg_error_type", data = list(...))` for classed conditions
- Never use `try()` in production — swallows context; prefer `tryCatch` with typed handlers
- `options(warn = 2)` in CI to promote warnings to errors

```r
safe_api_call <- function(url, max_retries = 3) {
  for (attempt in seq_len(max_retries)) {
    result <- tryCatch(
      httr2::request(url) |> httr2::req_perform(),
      error = function(e) {
        if (attempt == max_retries) rlang::abort("API unreachable", parent = e)
        Sys.sleep(min(2^attempt, 30))
        NULL
      }
    )
    if (!is.null(result)) return(result)
  }
}
```

## Linting & Profiling

- `lintr::lint_package()` in CI — zero lints before merge; configure `.lintr` for project rules
- `styler::style_pkg()` for auto-formatting; never mix manual + auto formatting
- `profvis::profvis()` to identify bottlenecks; vectorize inner loops before reaching for Rcpp
- `bench::mark()` for micro-benchmarks with memory allocation tracking

## Anti-Patterns

- ❌ `for` loops over data frame rows — use vectorized ops or `purrr::map`
- ❌ `T`/`F` instead of `TRUE`/`FALSE` — `T` can be overwritten by assignment
- ❌ `library()` inside functions — use `pkg::fn()` namespacing or Imports in DESCRIPTION
- ❌ `setwd()` anywhere — use `here::here()` or project-relative paths
- ❌ `attach()` or `<<-` — pollutes global env and causes hard-to-trace bugs
- ❌ `sapply()` in non-interactive code — return type varies; use `vapply()` or `purrr::map_*()`
- ❌ `source("utils.R")` chains — use package structure or `box::use()`
- ❌ Ignoring `R CMD check` warnings — treat as errors in CI
